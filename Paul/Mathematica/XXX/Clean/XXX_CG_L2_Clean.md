## Symbol reference — Section A (two-site chain setup)

### `ps[rep]`, `dim[rep]`, `J0[k][pat]`, `Jp[k][pat]`, `Jm[k][pat]`, `JJ[k,l,rep]`

- **Purpose:** Gelfand–Tsetlin (GT) basis machinery for a single $\mathfrak{gl}(N)$ irrep, and the
  matrix elements of its generators. Ported verbatim from `Experiments/XXX_su2.wb`; independent of
  $L$ (single-site building blocks reused by every later definition in this notebook).
- **Signature:**
  - `ps[rep_]` — `rep` is a rectangular-shape spec like `{λ,0}`; returns the list of GT patterns
    (basis vectors) for that irrep, memoized.
  - `dim[rep_]` — `Length[ps[{rep}]]`, the irrep dimension, memoized.
  - `J0[k][pat]`, `Jp[k][pat]`, `Jm[k][pat]` — Cartan/raising/lowering generator action on a single
    GT pattern `pat`, at GT-row level `k`, expressed as a formal sum of basis vectors `v[...]`.
  - `JJ[k_,l_,rep_]` — the full matrix of $E_{kl}$ (in the $\mathfrak{gl}(N)$ generator
    convention) on `rep`'s GT basis: diagonal ($k=l$), read off `Jp`/`Jm` coefficients ($l=k\pm1$),
    or built recursively from commutators of adjacent generators otherwise.
- **Returns:** `ps`/`dim` — pattern list / integer. `JJ` — a `dim[rep]`$\times$`dim[rep]` matrix.
- **Depends on:** nothing else in this notebook (pure GT combinatorics); recursion is internal to
  `JJ` itself.
- **Defined in:** Cell 3.
- **Paper reference:** underlies the "realized on the Gelfand–Tsetlin basis" statement in
  `paper/xxxCG.tex` §2 ("Lax matrix and transfer matrix"), but the paper doesn't spell out the GT
  formulas — this is standard representation theory, not a numbered paper equation.
- **Verified by:** no dedicated check cell; correctness is implicit in downstream checks (e.g. `L`
  reproducing the correct commutation relations via the transfer-matrix commutativity check, Cell
  12). Confirmed live in this notebook's kernel: `dim[{1,0}]=2`, `dim[{2,0}]=3`, `dim[{3,0}]=4`
  ($\dim V_\lambda=\lambda+1$ for $\mathfrak{gl}(2)$, as expected), and `JJ[1,1,{λ,0}]` gives the
  diagonal weight matrices for $\lambda=1,2,3$.
- **Example** (from live kernel state):
  ```wolfram
  JJ[1, 1, {2, 0}]
  (* {{2, 0, 0}, {0, 1, 0}, {0, 0, 0}} — diagonal J0 (weight) values 2,1,0 for the three λ=2 GT states *)
  ```

### `Ee[λ][i,j]`

- **Purpose:** spin-$\lambda/2$-representation generator matrix element $E_{ij}\in\mathrm{End}(V_\lambda)$,
  a thin wrapper over `JJ`.
- **Signature:** `Ee[λ_][i_,j_] := JJ[i,j,{λ,0}]`.
- **Returns:** `dim[{λ,0}]`$\times$`dim[{λ,0}]` matrix.
- **Depends on:** [`JJ`](#psrep-dimrep-j0kpat-jpkpat-jmkpat-jjklrep).
- **Defined in:** Cell 4.
- **Paper reference:** `paper/xxxCG.tex` §2, $E_{ij}\in\mathrm{End}(V_\lambda)$ ("the representation
  matrices of the $\mathfrak{gl}(2)$ generators", line following eq. for $L^{[\lambda]}$).

### `L[λ][i,j][u]`

- **Purpose:** single-site Lax matrix element, before any inhomogeneity shift is applied.
- **Signature:** `L[λ_][i_,j_][u_] := KroneckerDelta[i,j]*u*IdentityMatrix[dim[{λ,0}]] - h Ee[λ][j,i]`.
  $u$ is the (unshifted) spectral parameter; the inhomogeneity shift $u\to u-\theta$ is applied by
  the caller when the site is instantiated (see `L1`/`L2` below).
- **Returns:** the $(i,j)$ auxiliary-space matrix element of $L^{[\lambda]}(u)$, itself a
  `dim[{λ,0}]`$\times$`dim[{λ,0}]` operator on physical space.
- **Depends on:** [`Ee`](#eeλij), [`dim`](#psrep-dimrep-j0kpat-jpkpat-jmkpat-jjklrep).
- **Defined in:** Cell 5.
- **Paper reference:** `paper/xxxCG.tex` §2, eq.
  $L^{[\lambda]}(u)=u\,\mathbb{1}-h\sum_{i,j}E_{ji}\otimes e_{ij}$ — `L[λ][i,j][u]` is exactly the
  $e_{ij}$-coefficient of that expression, $u\delta_{ij}\mathbb{1}-hE_{ji}$.

### `G` (companion twist matrix)

- **Purpose:** the shared companion-form twist for the whole chain, with eigenvalues
  $\kappa_1,\kappa_2$ set numerically in Cell 1.
- **Signature:** `G = {{χ1,-χ2},{1,0}}` (not a function — a fixed numeric matrix once Cell 1 has
  run).
- **Depends on:** the global numeric parameters `χ1`, `χ2` from Cell 1 (`χ1=κ1+κ2`,
  `χ2=κ1κ2=1` here since `κ2=1/κ1`).
- **Defined in:** Cell 6 (numeric values from Cell 1).
- **Paper reference:** `paper/xxxCG.tex` §2.2 ("Companion twist and the fusion hierarchy"), eq.
  $G=\begin{pmatrix}\chi_1&-\chi_2\\1&0\end{pmatrix}$, $\chi_1=\kappa_1+\kappa_2$,
  $\chi_2=\kappa_1\kappa_2$.
- **Verified by:** `Experiments/Baxter_Solver_L2.wb` Cell 6 checks
  `{Tr[N[G]],Det[N[G]]} - {κ1+κ2,κ1 κ2}` via `Chop` and gets `{0,0}` (trace/det used rather than
  sorted eigenvalues, since $\kappa_2=\overline{\kappa_1}$ exactly and numeric eigenvalue
  extraction introduces $\sim10^{-17}$ real-part noise that can flip `Sort`'s tie-break — see that
  notebook's inline comment). Live kernel state: `G = {{0.7208...+0.I, -1}, {1, 0}}`, `χ1=0.7208...`,
  `χ2=1`, `θ1=1/3`, `θ2=1/7`, `h=1`.

### `L1[λ1,λ2][i,j][u]`, `L2[λ1,λ2][i,j][u]`

- **Purpose:** the two-site Lax operators on $V_{\lambda_1}\otimes V_{\lambda_2}$, each site's
  single-site Lax matrix (shifted by its own inhomogeneity) tensored with the identity on the
  other factor.
- **Signature:** `L1[λ1_,λ2_][i_,j_][u_] := KroneckerProduct[L[λ1][i,j][u-θ1], IdentityMatrix[dim[{λ2,0}]]]`
  (symmetric for `L2` with `θ2`/`λ2` swapped). Both `λ1` and `λ2` must be given explicitly even
  though only one enters the shifted `L[...]` call, because the identity factor's size depends on
  the *other* site's representation.
- **Returns:** `dim[{λ1,0}]*dim[{λ2,0}]`-square matrix.
- **Depends on:** [`L`](#lλiju), [`dim`](#psrep-dimrep-j0kpat-jpkpat-jmkpat-jjklrep); numeric `θ1`,
  `θ2` from Cell 1.
- **Defined in:** Cell 8.

### `T[λ1,λ2][i,j][u]`

- **Purpose:** the two-site monodromy matrix, $T(u)=L_1(u)\cdot L_2(u)$ in auxiliary space.
- **Signature:** `T[λ1_,λ2_][i_,j_][u_] := T[λ1,λ2][i,j][u] = Sum[L1[λ1,λ2][i,k][u].L2[λ1,λ2][k,j][u],{k,2}]`
  (memoized). Both factors carry the *same* spectral parameter $u$ — each site's inhomogeneity
  shift is already baked into `L1`/`L2`, unlike the single-site quantum-determinant construction
  which shifts the second factor by $h$ (see the Cell 9 markdown note: that shift is between two
  evaluations of the *same* site's $L$-matrix, which doesn't arise here since the two factors are
  different sites).
- **Depends on:** [`L1`, `L2`](#l1λ1λ2ijju-l2λ1λ2ijju).
- **Defined in:** Cell 10.

### `t2[λ1,λ2][1,1][u]`

- **Purpose:** the fundamental (companion-twist) transfer matrix for the two-site chain,
  $t_{1,1}(u)=\mathrm{tr}_a[G\,T(u)]$.
- **Signature:** `t2[λ1_,λ2_][1,1][u_] := Sum[T[λ1,λ2][i,j][u] G[[j,i]],{i,2},{j,2}]`.
- **Returns:** `dim[{λ1,0}]*dim[{λ2,0}]`-square matrix, a polynomial in $u$ of degree 2.
- **Depends on:** [`T`](#tλ1λ2ijju), [`G`](#g-companion-twist-matrix).
- **Defined in:** Cell 11.
- **Paper reference:** generalizes `paper/xxxCG.tex` §2 eq. $t_{1,1}(u)=\mathrm{tr}_a[G\,L^{[\lambda]}(u-\theta)]$
  from a single Lax matrix to the two-site monodromy $T$.
- **Verified by:** Cell 12 checks
  `t2[λ1,λ2][1,1][u].t2[λ1,λ2][1,1][v] - t2[λ1,λ2][1,1][v].t2[λ1,λ2][1,1][u]` over
  $\lambda_1,\lambda_2\in\{1,\dots,4\}$, `Simplify`d, `Flatten`ed, `Union`ed — designed to collapse
  to `{0}` (commutativity of the transfer matrix family at different spectral parameters, so its
  expansion coefficients are honest integrals of motion).

- **Purpose:** the same transfer matrix but with a formally *diagonal* twist $\kappa[1],\kappa[2]$,

- **Signature:** `t2diag[λ1_,λ2_][1,1][u_] := Sum[T[λ1,λ2][i,i][u] κ[i],{i,2}]`.
- **Depends on:** [`T`](#tλ1λ2ijju).

- **Gotcha:** `κ[1]`,`κ[2]` (function calls on the symbol `κ`) are **distinct symbols** from the

### `EE[λ1,λ2][i,j]`

- **Purpose:** the total (co-product) two-site $\mathfrak{gl}(2)$ generator,
  $E_{ij}\otimes\mathbb{1}+\mathbb{1}\otimes E_{ij}$.
- **Signature:** `EE[λ1_,λ2_][i_,j_] := KroneckerProduct[Ee[λ1][i,j],IdentityMatrix[dim[{λ2,0}]]] + KroneckerProduct[IdentityMatrix[dim[{λ1,0}]],Ee[λ2][i,j]]`.
- **Depends on:** [`Ee`](#eeλij), [`dim`](#psrep-dimrep-j0kpat-jpkpat-jmkpat-jjklrep).
- **Defined in:** Cell 14.
- **Verified by:** Cell 15 checks
  `EE[λ1,λ2][1,1]+EE[λ1,λ2][2,2]-(λ1+λ2)IdentityMatrix[...]` over $\lambda_1,\lambda_2\in\{1,\dots,4\}$,
  `Flatten`/`Union`ed — designed to collapse to `{0}`, confirming the central-charge identity
  (total particle-number operator on $V_{\lambda_1}\otimes V_{\lambda_2}$ equals $\lambda_1+\lambda_2$

### `tdiagasympt[λ1,λ2][u]`

### `qdetT[λ1,λ2][u]`

- **Purpose:** the (untwisted) quantum determinant of the two-site monodromy matrix.
- **Signature:** `qdetT[λ1_,λ2_][u_] := T[λ1,λ2][1,1][u].T[λ1,λ2][2,2][u-h] - T[λ1,λ2][2,1][u].T[λ1,λ2][1,2][u-h]`.
- **Depends on:** [`T`](#tλ1λ2ijju).
- **Defined in:** Cell 18.
- **Paper reference:** generalizes `paper/xxxCG.tex` §2.2 eq.
  $\mathrm{qdet}\,L^{[\lambda]}(u)=L^{[\lambda]}_{11}(u)L^{[\lambda]}_{22}(u-h)-L^{[\lambda]}_{21}(u)L^{[\lambda]}_{12}(u-h)$
  from the single-site Lax matrix to the two-site monodromy $T$.

### Fusion hierarchy — `t2[λ1,λ2][0,s][u]`, `t2[λ1,λ2][a,0][u]`, `t2[λ1,λ2][2,1][u]`, `t2[λ1,λ2][1,s][u]`

### `ν1[λ1,λ2][u]` — Yangian weight eigenvalue of $T_{11}$ on the highest-weight state

- **Purpose:** added 2026-07-16 to fix a normalization bug in `x2` (see below): the eigenvalue that
  the two-site monodromy element $T_{11}(u)$ takes on the two-site highest-weight state, with
  **each site's own** representation label shifting **only its own** inhomogeneity factor. It
  replaces the earlier use of the symmetric `Qθ[u]=(u-θ1)(u-θ2)` evaluated at a single argument
  shifted by $-\lambda_1 h$ — which silently applied *site 1's* shift to *both* factors, including
  the $\theta_2$ one that should have carried site 2's own $\lambda_2$.
- **Signature:** `ν1[λ1_,λ2_][u_] := (u-θ1-h λ1)(u-θ2-h λ2)`.
- **Returns:** a scalar, degree-2 polynomial in $u$.
- **Depends on:** nothing else project-defined.
- **Defined in:** Cell 25.
- **Why the fix has to live inside the function, not the call site:** the bug in the old `x2`
  denominator came from faking a per-site shift by shifting the *argument* of an already-symmetric
  fixed function (`Qθ(θ_i+(k-1-\lambda_1)h)` for *both* $i=1,2$) instead of giving each factor its
  own site-specific shift from the start. `ν1[λ1,λ2][u]` fixes this by construction: its two
  factors are independently parametrized by $\lambda_1,\lambda_2$, so there is no argument-shift
  trick left to get wrong. Used directly by `x2`'s denominator (below) and, via
  `aFun[λ1,λ2][u] := ν1[λ1,λ2][u]`, by Section B's Baxter-equation coefficient (Section B).
- **Gotcha:** a *different*, single-site (arity-1) version `ν1[λ_][u_] := (u-θ1-hλ)` is defined
  independently in Section E — same symbol name, distinguished only by argument count (Wolfram
  dispatches on the pattern `[λ1_,λ2_][u_]` vs. `[λ_][u_]` as separate DownValues on the same
  head), not a conflict, but worth knowing when searching the notebook for `ν1`.

## Symbol reference — Section B (Baxter/TQ solver)

The routines below jointly solve $t_{1,1}(u)\Psi = \tau(u)\Psi$ and the Baxter TQ equation
$$\kappa_1\,a(u)\,q(u+h) + \kappa_2\,Q_\theta(u)\,q(u-h) = \tau(u)\,q(u)$$
for the two-site chain, in two stages: (1) `TauEigensystem` diagonalizes the *genuine* transfer
matrix directly (no oversized Baxter-matrix proxy, no post-hoc spurious-root filtering) to get
every $(M,n)$-indexed eigenvalue $\tau=\tau_0+J_n$; (2) `QSolve` then solves for the monic
degree-$M$ Q-function $q_n(u)$ as a linear `NullSpace` problem at the now-known $J_n$. This
supersedes an earlier (superseded, still in `Experiments/`) approach that diagonalized an
oversized $(M{+}1)$-dimensional Baxter matrix directly and could manufacture spurious extra roots.
See `Experiments/Baxter_Solver_L2.wb` for the full derivation writeup and validation/regression
suite (not duplicated into this Clean notebook). Section C below builds a full numeric verification
of orthogonality of the resulting `Q1` Q-functions; Section F builds a closed-form dictionary
relating them, functionally, to an operatorial overlap.

### `aFun[λ1,λ2][u]`, `τ0[λ1,λ2,M][u]`, `Qθ[u]` (Section B), `qpoly[clist,uu]`

- **Purpose:** the Baxter-equation ingredients, defined locally so Section B is self-contained: the
  Bethe/Baxter "$a(u)$" coefficient, the $J$-independent part of the transfer-matrix eigenvalue
  $\tau_0$, the two-site $Q_\theta$ factor (redefinition of Section A's, see gotcha above), and a
  helper that evaluates an ascending-coefficient-list polynomial without an indeterminate $0^0$.
- **Signature:**
  - `aFun[λ1_,λ2_][u_] := ν1[λ1,λ2][u]` — **deduped 2026-07-16:** `aFun` used to carry its own
    literal copy of the formula `(u-θ1-hλ1)(u-θ2-hλ2)`; it is now a plain alias for Section A's
    two-site `ν1` (same Yangian-weight-eigenvalue object `x2`'s denominator uses), so there is a
    single source of truth for this formula instead of two copies that could silently drift apart.
    The numeric value is unchanged by this dedup.
  - `Qθ[u_] := (u-θ1)(u-θ2)` — identical to Section A's, re-`ClearAll`'d and redefined.
  - `τ0[λ1_,λ2_,M_][u_] := (κ1+κ2)u^2 - (θ1+θ2)(κ1+κ2)u - u h(κ1(λ1+λ2-M)+κ2 M)` — note zero
    constant ($u^0$) term by construction, so the full eigenvalue is $\tau=\tau_0+J$ with $J$ the
    constant term.
  - `qpoly[clist_,uu_] := Module[{x}, (clist.x^Range[0,Length[clist]-1]) /. x -> uu]` — evaluates
    a polynomial given ascending coefficients `{c0,...,cM}` at `uu`.
- **Depends on:** [`ν1`](#ν1λ1λ2u--yangian-weight-eigenvalue-of-t11-on-the-highest-weight-state)
  (Section A).
- **Defined in:** Cell 30.
- **Paper reference:** none — `paper/xxxCG.tex` does not yet state a Baxter/TQ equation (its SoV
  section stops at the pseudovacuum/covector construction). `a(u)=\prod_i(u-\theta_i-h\lambda_i)$,
  $Q_\theta(u)=\prod_i(u-\theta_i)$ is the standard XXX Baxter-equation form generalized to two
  inhomogeneous sites with a companion twist of eigenvalues $\kappa_1,\kappa_2$.

### `BaxterMatrix[λ1,λ2,M]`

- **Purpose:** the fixed linear operator, in the monic-degree-$M$-polynomial coefficient basis,
  whose nullspace at a known $J$ gives the Q-function's coefficients. Historically also used
  (superseded approach, in `Experiments/`) as an oversized eigenvalue problem; here it is used only
  as a linear operator via `QSolve`.
- **Signature:** `BaxterMatrix[λ1_,λ2_,M_]` builds, for $k=0,\dots,M$, the column
  $O[u^k] = \kappa_1\,a(u)(u+h)^k + \kappa_2\,Q_\theta(u)(u-h)^k - \tau_0(u)\,u^k$, expands it as a
  polynomial in $u$, and hard-asserts (`Abort[]` on failure) that the degree-$(M{+}1)$ and
  $(M{+}2)$ coefficients vanish for every column — this is where a wrong `aFun`/`Qθ`/`τ0` or a
  wrong $ee_1=\lambda_1+\lambda_2-M$ convention would show up. The surviving degree-$0..M$
  coefficients of each column become the matrix's columns.
- **Returns:** an $(M{+}1)\times(M{+}1)$ matrix.
- **Depends on:** [`aFun`, `τ0`, `Qθ`](#afunλ1λ2u-τ0λ1λ2mu-qθu-section-b-qpolyclistuu).
- **Defined in:** Cell 31.
- **Verified by:** the hard in-function assert (no separate check cell in this notebook); Cell 31's
  smoke test `Dimensions /@ (BaxterMatrix[1,1,#] & /@ {0,1,2})` is designed to give
  `{{1,1},{2,2},{3,3}}`. Confirmed via the predecessor `Experiments/Baxter_Solver_L2.wb` (Cells 4–5,
  identical code).

- **Signature:** `t2Coeffs[λ1_,λ2_] := t2Coeffs[λ1,λ2] = Module[...]` (memoized), returns

### `dPred[M,λ1,λ2]`

- **Purpose:** predicted dimension of the weight-$M$ subspace of $V_{\lambda_1}\otimes V_{\lambda_2}$
  — the number of $(m_1,m_2)$ pairs with $m_1+m_2=M$ fixed and $0\le m_i\le\lambda_i$. This is the
  exact multiplicity `TauEigensystem` (and `TauEigensystemLeft`, Section D) must reproduce per
  $M$-sector (no more, no fewer).
- **Signature:** `dPred[M_,λ1_,λ2_] := Min[M,λ1,λ2,λ1+λ2-M] + 1`.
- **Defined in:** Cell 33.
- **Depends on:** nothing (pure combinatorics).

### `TauEigensystem[λ1,λ2]`

- **Purpose:** diagonalizes the *genuine* transfer matrix $t_{1,1}[\lambda_1,\lambda_2](u)$
  directly (not the oversized Baxter-matrix proxy), returning every $(M,n)$-indexed eigenstate with
  a hard genuineness residual check.
- **Signature:** `TauEigensystem[λ1_,λ2_] := TauEigensystem[λ1,λ2] = Module[...]` (memoized).
  Algorithm: since $C_0,C_1,C_2$ pairwise commute, diagonalizes a generic combination
  $C_0+rC_1$ (trying $r\in\{7/3,\pi,e,11/5,13/4,\sqrt2\}$ until the combination's eigenvectors have
  full rank $d=(\lambda_1{+}1)(\lambda_2{+}1)$ and $d$ distinct eigenvalues — guards against an
  accidental degeneracy at a specific $r$). For each eigenvector $v$: extracts
  $t_0=\frac{v\cdot C_0 v}{v\cdot v}$, $t_1=\frac{v\cdot C_1 v}{v\cdot v}$, and hard-`Abort`s if the
  residuals $\|C_0v-t_0v\|,\|C_1v-t_1v\|$ exceed $10^{-15}$ (this residual check *is* the
  genuineness test, replacing the old post-hoc spurious-state filter). $M$ is read off
  algebraically from $t_1$ via
  $$M=\frac{-(\theta_1+\theta_2)(\kappa_1+\kappa_2)-h\kappa_1(\lambda_1+\lambda_2)-t_1}{h(\kappa_2-\kappa_1)}$$
  (inverting $\tau_0$'s $u^1$ coefficient), hard-asserted to land within $10^{-10}$ of an integer;
  $J=t_0$ directly (since $\tau_0$'s $u^0$ coefficient is zero by construction). States within each
  $M$-sector are sorted ascending by $(\mathrm{Re}[J],\mathrm{Im}[J])$ to assign $n$, and each
  sector's count is hard-asserted to equal `dPred[M,λ1,λ2]` exactly.
- **Returns:** an `Association` keyed by `{M,n}` $\to$ `<|"J"->J_n, "Psi"->eigenvector normalized to
  last component 1|>`.
- **Depends on:** [`t2Coeffs`](#t2coeffsλ1λ2), [`dPred`](#dpredmλ1λ2).
- **Defined in:** Cell 33.
- **Paper reference:** none (Baxter/TQ machinery not yet in `paper/xxxCG.tex`).
- **Verified by:** the in-function hard asserts (rank/degeneracy resolution, residual $<10^{-15}$,
  integer $M$ within $10^{-10}$, per-sector count exactly matching `dPred`) constitute the check;
  a real failure aborts rather than silently returning wrong data. Independently,
  `Experiments/Baxter_Solver_L2.wb` Cells 10–11 run an *external* re-check for
  $(\lambda_1,\lambda_2)=(1,1)$ and $(2,1)$, both "STAGE-1 VALIDATION ... PASSED". This notebook's
  own live kernel (confirmed via `wolfbook_getKernelState`) has `TauEigensystem[1,1]` through
  `TauEigensystem[1,3]` (and further reps referenced by Sections C–F, e.g. `{2,2}`,`{3,1}`)
  memoized live.
- **Example** (from live kernel state):
  ```wolfram
  KeySort[TauEigensystem[1, 1]][[All, "J"]] // N
  (* <|{0,0} -> 0.5664055180884988+1.3769645669711723 I,
  ```

### `Psi[λ1,λ2][M,n]`, `τ[λ1,λ2][M,n][u]`

- **Purpose:** bounds-checked, on-demand accessors over the shared `TauEigensystem` cache. `Psi`
  returns the fixed (last-component-normalized) eigenvector — the right eigenvector paired against

- **Signature:** `Psi[λ1_,λ2_][M_,n_]`, `τ[λ1_,λ2_][M_,n_][u_]`. Both require $M$ an integer in
  $0..\lambda_1+\lambda_2$ and $n$ an integer in $0..M$ (else `Message[·::bnds]` and `$Failed`); if
  $n$ is in the gross $0..M$ range but exceeds the *actual* weight multiplicity
  `dPred[M,λ1,λ2]-1`, fires `Message[·::spurious]` and returns `Missing["NoState",...]`.
- **Depends on:** [`TauEigensystem`](#taueigensystemλ1λ2), [`dPred`](#dpredmλ1λ2),
  [`τ0`](#afunλ1λ2u-τ0λ1λ2mu-qθu-section-b-qpolyclistuu).

### `QSolve[λ1,λ2][M,n]`

- **Purpose:** solves the Baxter TQ equation as a *linear* problem (not an eigenvalue problem, since
  $J_n$ — hence $\tau$ — is already known from `TauEigensystem`): finds the monic degree-$M$
  polynomial $q_n(u)$ with `(BaxterMatrix[λ1,λ2,M] - Jn·Id).q_n = 0`.
- **Signature:** `QSolve[λ1_,λ2_][M_,n_] := QSolve[λ1,λ2][M,n] = Module[...]` (memoized). Same
  bounds/spurious checks as `Psi`/`τ`. Builds `mat = BaxterMatrix[λ1,λ2,M] - Jn·IdentityMatrix[M+1]`
  (singular by construction) and calls
  `NullSpace[Chop[mat, 10^-12], Tolerance -> 10^-10]`.
- **Returns:** the length-$(M{+}1)$ nullspace vector normalized so its last (top, $u^M$) coefficient
  is $1$ (monic), or `$Failed`/message on `degnull`/`lastzero`/`bnds`/`spurious` failure.
- **Depends on:** [`BaxterMatrix`](#baxtermatrixλ1λ2m), [`TauEigensystem`](#taueigensystemλ1λ2),
  [`dPred`](#dpredmλ1λ2).
- **Defined in:** Cell 35.
- **Precision gotcha (from the notebook's own comment):** `BaxterMatrix[λ1,λ2,M]-Jn·Id` is
  genuinely singular, but its would-be-zero entries come out as **precision-tracked** zeros. `Chop`
  the matrix (threshold $10^{-12}$) **before** `NullSpace` — see the "Known gotchas" section below.
- **Verified by:** `Experiments/Baxter_Solver_L2.wb` Cell 13 is an explicit regression guard for
  $(\lambda_1,\lambda_2,M,n)=(2,1,0,0)$ — "REGRESSION GUARD PASSED". End-to-end correctness of the
  resulting $q_n$ is confirmed by the stage-2 checks under [`Q1`](#q1λ1λ2mnu) below, and Section C's
  Gram-matrix orthogonality check.
- **Example** (live kernel state, `QSolve[2,1][2,0]` memoized):
  ```wolfram

      1.00000000000000000000000000+0.×10^-27 I} *)
  ```

## Symbol reference — Section C (FSoV scalar product / orthogonality)

Section C ports the Functional Separation of Variables (FSoV) machinery from
`Experiments/FSoV_L2.wb` and uses it to numerically verify orthogonality/non-degeneracy of the
`Q1` Q-functions computed in Section B, over a full representation at once (all $M$-sectors,
including cross-$M$ pairs). See
`docs/superpowers/specs/2026-07-16-fsov-scalar-product-design.md` for the design writeup.

### `mu[λ1,λ2][i][u]`

- **Purpose:** the FSoV measure for site $i\in\{1,2\}$: a rational function with simple poles on
  that site's inhomogeneity tower $\theta_i+hk$, $k=0,\dots,\lambda_i$, carrying an extra
  cross-site oscillatory factor that vanishes the residue sum of `1` (needed for `br` to correctly
  isolate the $\theta_i$-tower contribution without a spurious constant term).
- **Signature:**
  `mu[λ1_,λ2_][1][u_] := Nn[λ1,λ2][1]/(Product[(u-θ1-h k),{k,0,λ1}]Product[(u-θ2-h k),{k,0,λ2}])(1-Exp[2 π I/h (u-θ2)])`,
  symmetric for site 2 with `θ1`/`θ2` swapped in the oscillatory factor.
- **Returns:** a rational (times exponential) function of $u$.
- **Depends on:** [`Nn`](#nnλ1λ2i-nnλ1λ2i).
- **Defined in:** Cell 38.
- **Paper reference:** none yet — `paper/xxxCG.tex` does not state an FSoV scalar product; ported
  from the exploratory `Experiments/FSoV_L2.wb`.

### `Nn[λ1,λ2][i]`, `nn[λ1,λ2][i]`

- **Purpose:** normalization factors in the site-$i$ measure, fixing its overall residue sum so that
  `det` below has the right normalization to be a genuine scalar product.
- **Signature:**
  `Nn[λ1_,λ2_][1] := 1/(2 π I)/(1-Exp[2 π I/h(θ1-θ2)]) nn[λ1,λ2][1]`;
  `nn[λ1_,λ2_][1] := (-1)^(λ1+λ2-1)h^(λ1)Gamma[λ1+1] Product[θ2+k h-θ1,{k,0,λ2}]//Simplify`
  (symmetric for site 2).
- **Depends on:** nothing else project-defined.
- **Defined in:** Cell 39.

### `br[λ1,λ2][i][f]`

- **Purpose:** the bracket pairing for site $i$: the residue sum of $f\cdot\mu_i$ over all of site
  $i$'s poles — a contour integral around the whole $\theta_i$-tower.
- **Signature:**
  `br[λ1_,λ2_][1][f_] := 2 π I Plus@@(Residue[f mu[λ1,λ2][1][u],{u,#}]&/@Table[θ1+h k,{k,0,λ1}])//Expand//Simplify`
  (symmetric for site 2). `f` is any function of `u` (typically a product of Q-functions).
- **Returns:** a symbolic/numeric value (whatever `f` evaluates to under the residue sum).
- **Depends on:** [`mu`](#muλ1λ2iu).
- **Defined in:** Cell 40.

### `det[λ1,λ2][f]`

- **Purpose:** the $2\times2$ Wronskian-type determinant built from `br[·][f]` and `br[·][u f]` at
  both sites — this **is** the FSoV/SoV scalar product. For $f=Q_A(u)Q_B(u)$ (a product of two
  Baxter Q-functions), `det[λ1,λ2][f]` realizes $\langle Q_A,Q_B\rangle$: it must vanish for $A\neq
  B$ and be nonzero for $A=B$.
- **Signature:**
  `det[λ1_,λ2_][f_] := 1/(f/.u:>θ1)/(f/.u:>θ2)/(θ2-θ1)({{br[λ1,λ2][1][f],br[λ1,λ2][1][u f]},{br[λ1,λ2][2][f],br[λ1,λ2][2][u f]}}//Det//Expand//Simplify)`.
- **Returns:** a scalar (the normalized $2\times2$ determinant of bracket values).
- **Depends on:** [`br`](#brλ1λ2if).
- **Defined in:** Cell 41.
- **Verified by (port-fidelity check):** Cell 42's `fsovL2CachedFormula` is a verbatim transcription
  of `FSoV_L2.wb`'s own cached closed form for `det[1,1][f[u] z^(u/h)]` with $\theta_1,\theta_2,h$
  symbolic; `portCheckDiff = Simplify[det[1,1][f[u] z^(u/h)] - fsovL2CachedFormula[θ1,θ2,h,z,f]]`
  confirms the port is exact — live kernel state gives `portCheckDiff = 0`.

### `statesOf[λ1,λ2]`

- **Purpose:** the sorted `{M,n}` key list for a representation, read off `TauEigensystem`'s
  memoized cache (Section B) rather than re-deriving `dPred` combinatorics here. Reused throughout
  Sections C–F (`GramMatrix`, biorthogonality checks, `alphaTab`, the rep-sweep validation).
- **Signature:** `statesOf[λ1_,λ2_] := SortBy[Keys[TauEigensystem[λ1,λ2]], {First,Last}]`.
- **Returns:** a list of `{M,n}` pairs, sorted first by $M$ then by $n$.
- **Depends on:** [`TauEigensystem`](#taueigensystemλ1λ2).
- **Defined in:** Cell 43.
- **Example** (live kernel state):
  ```wolfram
  states21 = statesOf[2, 1]
  (* {{0,0},{1,0},{1,1},{2,0},{2,1},{3,0}} *)
  ```

### `GramMatrix[λ1,λ2]`, `G21`

- **Purpose:** the full Gram matrix $G_{(M,n),(M',n')} = \det[\lambda_1,\lambda_2]\big[Q_1(M,n)(u)\,
  Q_1(M',n')(u)\big]$ over **all** states of a representation at once — covering both same-$M$
  degeneracy and cross-$M$ pairs in a single object. `G21` is the concrete instance for
  $(\lambda_1,\lambda_2)=(2,1)$ ($d=6$, weight sectors $M=0,1,2,3$ with `dPred[M,2,1]=1,2,2,1`, so
  $M=1,2$ are genuinely degenerate 2-dimensional sectors — a nontrivial test of the scalar
  product's ability to resolve degenerate states).
- **Signature:** `GramMatrix[λ1_,λ2_] := Module[{s=statesOf[λ1,λ2]}, Table[det[λ1,λ2][Q1[λ1,λ2][s1[[1]],s1[[2]]][u] Q1[λ1,λ2][s2[[1]],s2[[2]]][u]],{s1,s},{s2,s}]]`;
  `G21 = GramMatrix[2,1]` (Cell 45), a $6\times6$ matrix.
- **Depends on:** [`det`](#detλ1λ2f), [`statesOf`](#statesofλ1λ2), [`Q1`](#q1λ1λ2mnu).
- **Defined in:** Cell 43 (`GramMatrix`), Cell 45 (`G21`).
- **Paper reference:** none — the FSoV scalar product/orthogonality result has no counterpart yet
  in `paper/xxxCG.tex`.
- **Verified by:** Cell 47's hard-assert check (own message names `FSoVGram::notorthogonal`,
  `FSoVGram::degenerate`): computes `maxOffDiag` over all 30 off-diagonal entries of the $6\times6$
  matrix (same-$M$ and cross-$M$ pairs alike) and `minDiagAbs` over the 6 diagonal entries, aborting

## Symbol reference — Section D (left eigenvectors and biorthogonality)

Section D is an **independent** construction (it never edits `TauEigensystem`): it diagonalizes
`Transpose[C0 + r C1]` for its own generic $r$, then matches each resulting left eigenvector to an
existing $\{M,n\}$ state by comparing eigenvalues. It exists to build $\langle\Psi|\Psi\rangle$ as a
genuine bilinear (no conjugation) pairing and compare it against Section C's FSoV scalar product
`det[Q,Q]`. See `docs/superpowers/specs/2026-07-16-fsov-left-eigenvector-ratio-design.md`; this is
exploratory in the sense that no predicted formula for `normRatio` exists yet in
`paper/xxxCG.tex` — the cell reports the ratio, it doesn't assert a value for it (though the
observed value is consistently $1$).

### `TauEigensystemLeft[λ1,λ2]`

- **Purpose:** diagonalizes the transpose of the same generic combination `C0+rC1` used by
  `TauEigensystem` (for its own, independently chosen $r$), matches each left eigenvector to an
  `{M,n}` state via its combo-eigenvalue, and returns it fixed to last-component $1$.
- **Signature:** `TauEigensystemLeft[λ1_,λ2_] := TauEigensystemLeft[λ1,λ2] = Module[...]` (memoized).
  Algorithm: picks $r$ from candidates `{17/6, π+1, e+1, 9/4, 19/6, √3}` such that
  `Transpose[C0+rC1]` has full rank $d$ and $d$ distinct eigenvalues (`TauEigensystemLeft::rankdef`
  on failure). For each state `{M,n}` in `statesOf[λ1,λ2]`, the target combo-eigenvalue is computed
  algebraically as $\tau_{M,n}(r)-(\kappa_1+\kappa_2)r^2$ (since $C_2=(\kappa_1+\kappa_2)\mathbb{1}$
  exactly and $t_{1,1}(r)\Psi_{M,n}=\tau_{M,n}(r)\Psi_{M,n}$ for the *right* eigenvector), and the
  closest left eigenvalue is matched to it — hard-asserting the match distance is below $10^{-10}$
  (`::nomatch`), that no left-eigenvector index is claimed twice (`::dupmatch`), and that its last
  component isn't vanishingly small before normalizing (`::lastzero`). The total state count is
  hard-asserted to equal $d=(\lambda_1{+}1)(\lambda_2{+}1)$ (`::countmismatch`).
- **Returns:** an `Association` keyed by `{M,n}` $\to$ left eigenvector, normalized to last
  component $1$.
- **Depends on:** [`t2Coeffs`](#t2coeffsλ1λ2), [`statesOf`](#statesofλ1λ2), [`τ`](#psiλ1λ2mn-τλ1λ2mnu).
- **Defined in:** Cell 49.
- **Verified by:** the in-function hard asserts constitute the check (as with `TauEigensystem`).
  Cell 50's smoke test confirms `KeySort[TauEigensystemLeft[2,1]]//Keys` reproduces the same 6 keys

## Symbol reference — Section E (exact operatorial factorization of the L=2 SoV basis)

Section E builds a genuine **standalone** length-1 (single-site) fused transfer-matrix hierarchy and
SoV covector — the paper's $L=1$ construction, built here entirely independently of Section A's
two-site `t2`/`x2` (it reuses only the single-site `L`, `G`, `xSingle`, `dim`) — and uses it
to prove that the $s_2=0$ slice of the two-site SoV basis `x2` **factorizes exactly** as
`x2[λ1,λ2][{s1,0}] = xL1[λ1][{s1}] ⊗ xSingle[λ2][{0}]`, with **no leftover scalar** (ratio
bit-exactly $1$). Sections A–D are read-only from this point on. See
`docs/superpowers/specs/2026-07-16-xxx-sov-factorization-functional-overlap-design.md`.

**Rewritten 2026-07-16 alongside the `ν1`/`x2` normalization fix (Section A):** an earlier version
of this section found what looked like a nontrivial rank-1 "site-2 passthrough eigenvalue"
`μSite2`/`cPredCorr` multiplying the naive tensor product. That finding was **an artifact of the
same `x2` normalization bug** described in Section A's `ν1` entry — `x2`'s old denominator divided
by the symmetric `Qθ(u)=(u-θ1)(u-θ2)` evaluated at a single argument shifted by $-\lambda_1 h$,
which incorrectly applied site 1's shift to *both* factors, including the $\theta_2$ one that
should have carried site 2's own $\lambda_2$. Once `x2` divides by the corrected `ν1[λ1,λ2][u]`
instead (each factor shifted by its own site's $\lambda$), the ratio comes out **exactly** $1$ for
every swept case, not merely a constant — i.e. there is no genuine extra site-2 eigenvalue beyond
what `ν1` already encodes; the old `cPredCorr`/`μSite2` "finding" was bookkeeping fallout from the
bug, not new physics. The `cPredCorr` symbol has been removed entirely from the notebook (the
factorization check now asserts the ratio equals $1$ directly, not a nontrivial closed form); the
`μSite2` computation is *kept*, relabeled, as a genuine but no-longer-mysterious operator identity
(see below) — it is exactly the $\lambda_2$-dependent factor that `ν1` already factors into.

## Symbol reference — Section F (functional CG-overlap dictionary)

Section F is the notebook's headline result: a closed dictionary equating a purely *functional*
object built from the numeric Baxter Q-functions (`FLdet`, a 1-site FSoV determinant of
$k^{u/h}Q_1$) with a purely *operatorial* object built from rotating the SoV pseudovacuum by a
one-parameter unipotent group element (`OPg`), via a single linear rescaling of the parameter,
$\varphi(k)=\alpha k$ with $\alpha=-1$ found universally over every representation and state tested.
See `docs/superpowers/specs/2026-07-16-xxx-sov-factorization-functional-overlap-design.md`.

### `Nn1[λ]`, `mu1[λ][u]`, `br1[λ][f]`

- **Purpose:** the length-1 reduction of Section C's FSoV measure/bracket, stripped of all site-2
  factors and the cross-site oscillatory term, and **re-normalized** (not simply restricted) so that
  the leading ($\theta_1$-node) weight is exactly $1$: `br1[λ][f] = f(θ1) + (higher-tower terms)`.
  `Nn1` is *derived*, not guessed — by forcing $w_0 := 2\pi i\,\mathrm{Res}[\mathrm{mu1},\theta_1]=1$.
- **Signature:**
  `Nn1[λ_] := Nn1[λ] = Module[{uu}, 1/(2 π I Residue[1/Product[(uu-θ1-h k),{k,0,λ}],{uu,θ1}])]`
  (memoized); `mu1[λ_][u_] := Nn1[λ]/Product[(u-θ1-h k),{k,0,λ}]`;
  `br1[λ_][f_] := 2 π I Plus@@(Residue[f mu1[λ][u],{u,#}]&/@Table[θ1+h k,{k,0,λ}])`.
- **Returns:** `Nn1` — a scalar; `mu1` — a rational function of $u$; `br1` — whatever `f`'s residue
  sum evaluates to.
- **Depends on:** nothing else project-defined (self-contained single-site construction).
- **Defined in:** Cell 64.
- **Gotcha:** for $\lambda\ge1$, the residues of `mu1` alone sum to $0$ (so `br1[λ][1]=0` — a
  rational measure with no residue at infinity), so the correct normalization statement to check is
  $w_0=1$, not `br1[λ][1]=1`.
- **Verified by:** Cell 65 hard-asserts, for $\lambda=1,2,3$: the leading weight
  `w1[λ][[1]] == 1` exactly (`Nn1Check::badlead` otherwise) and `br1[λ][1] == 0` exactly
  (`Nn1Check::badsum` otherwise). Live kernel state: `Nn1[1]=(I/2)/Pi`, `Nn1[2]=-I/Pi`,
  `Nn1[3]=(3I)/Pi`. Cell 66 additionally sanity-checks that `br1` acting on a generic symbolic
  function `g[u]` reproduces the weighted tower sum $\sum_j w_j\,g(\theta_1+hj)$ exactly (`Simplify`
  to $0$), confirming `br1` is honestly the weighted residue sum it claims to be, not just correct
  on its leading term.

### `det1[λ1][f]`, `FLdet[λ1,λ2][M,n][k]`

- **Purpose:** the 1-site FSoV **determinant** (not the bare bracket) — the 1-site reduction of
  Section C's `det[λ1,λ2][f]`, which divides by $f$ at *both* nodes $\theta_1,\theta_2$; only the
  $\theta_1$-node survives at 1 site. Dividing by $f(\theta_1)$ cancels the fractional prefactor
  $(k\kappa_1)^{\theta_1/h}$ that `k^(u/h) Q1[...]` carries (since `Q1` itself carries
  $\kappa_1^{u/h}$), leaving a clean polynomial in $k$. `FLdet` is this determinant applied to
  `k^(u/h) Q1[λ1,λ2][M,n][u]`.
- **Signature:** `det1[λ1_][f_] := br1[λ1][f]/(f/.u->θ1)`;
  `FLdet[λ1_,λ2_][M_,n_][k_] := det1[λ1][k^(u/h) Q1[λ1,λ2][M,n][u]]`.
- **Returns:** `FLdet[λ1,λ2][M,n][k]` is a degree-$\lambda_1$ polynomial in $k$ with constant term
  exactly $1$ (a consequence of $w_0=1$).
- **Depends on:** [`br1`](#nn1λ-mu1λu-br1λf), [`Q1`](#q1λ1λ2mnu).
- **Defined in:** Cell 70.
- **Paper reference:** none — this is the notebook's own novel functional CG-overlap object.
- **Verified by:** Cell 70's own smoke test, `FLdet[1,2][0,0][k]//Simplify`, designed to be a
  degree-1 polynomial in $k$ with constant term exactly $1$ and no fractional power of $k$
  (confirming the prefactor cancellation). Full correctness (agreement with `OPg` under
  $\varphi(k)=\alpha k$) is established by Cells 71–73 below.

### `gRot[λ][a,b][φ]`, `OP[λ1,λ2][a,b][M,n][φ]`, `genNontrivial`, `OPg[λ1,λ2][M,n][φ]`

- **Purpose:** the operatorial side of the dictionary. `gRot` is the finite unipotent group element
  $g=\exp[\varphi E_{ab}]$, built as an **exact truncated series** (not `MatrixExp`, which would
  return an unevaluated symbolic exponential here) since $E_{ab}$ is nilpotent of order
  $\le\lambda+1$ on the finite-dimensional rep. `OP` rotates the site-1 pseudovacuum covector by
  `gRot`, tensors with the site-2 pseudovacuum, and contracts with the two-site right eigenvector
  `Psi` — the operatorial CG overlap. `genNontrivial` picks out which single-site generator
  ($E_{12}$ or $E_{21}$) actually produces $\varphi$-dependence (the lowest-weight covector
  $\langle x_0|$ annihilates the other one), and `OPg` fixes downstream code to that choice.
- **Signature:**
  `gRot[λ_][a_,b_][φ_] := IdentityMatrix[dim[{λ,0}]] + Sum[φ^m/m! MatrixPower[Ee[λ][a,b],m],{m,1,λ}]`;
  `OP[λ1_,λ2_][a_,b_][M_,n_][φ_] := Flatten[KroneckerProduct[xSingle[λ1][{0}].gRot[λ1][a,b][φ], xSingle[λ2][{0}]]] . Psi[λ1,λ2][M,n]`;
  `genNontrivial = {2,1}` (fixed once, from the Cell 68 test — see below);
  `OPg[λ1_,λ2_][M_,n_][φ_] := OP[λ1,λ2][genNontrivial[[1]],genNontrivial[[2]]][M,n][φ]`.
- **Depends on:** [`Ee`](#eeλij), [`dim`](#psrep-dimrep-j0kpat-jpkpat-jmkpat-jjklrep),
  [`xSingle`](#xsingleλ0-x2λ1λ2s1s2), [`Psi`](#psiλ1λ2mn-τλ1λ2mnu).
- **Defined in:** Cell 67 (`gRot`, `OP`), Cell 68 (`genNontrivial`, via a test), Cell 69 (`OPg`).
- **Gotcha (kernel quirk, noted in the notebook's own comment, not a typo):**
  `MatrixPower[singularMatrix,0]` does **not** auto-reduce to `IdentityMatrix` in this kernel (it
  fires `MatrixPower::sing` and stays unevaluated) even though $m=0$ is a concrete nonnegative
  integer — `Ee[λ][a,b]` is singular/nilpotent for the generators used here. `gRot`'s $m=0$ identity
  term is therefore split off explicitly and `MatrixPower` is only summed over $m=1,\dots,\lambda$.
- **Verified by:** Cell 68 tests `OP[2,1][1,2][1,0][φ]` (generator $E_{12}$) vs.
  `OP[2,1][2,1][1,0][φ]` (generator $E_{21}$) on the fixed case $(\lambda_1,\lambda_2)=(2,1)$,
  state $(1,0)$: live kernel state shows `op12` is exactly $\varphi$-independent
  ($1.+0.\times I$, constant) while `op21` genuinely depends on $\varphi$ — confirming
  `genNontrivial={2,1}` ($E_{21}$) is the correct, nontrivial choice. Cell 69's smoke test confirms
  `Exponent[OPg[·][φ],φ] <= λ1` for a $\lambda_1=1$ and a $\lambda_1=2$ case.

### `αVal`, `φuniversal[k]` — the closed dictionary

- **Purpose:** the notebook's headline discovery: the same linear rescaling $\varphi(k)=\alpha k$,

- **`κ[1]`/`κ[2]` vs `κ1`/`κ2`:** Cell 13's `t2diag` and Cell 16's `tdiagasympt` use the formal,
  purely symbolic indexed function `κ[i]` — these are unrelated to the numeric plain variables
  `κ1`,`κ2` set in Cell 1 (the actual companion-twist eigenvalues used everywhere else, including
  all of Sections B–F). They only appear together, symbolically, in the Cell 17 asymptotics
  cross-check.
- **`ν1` defined twice with different arity (added 2026-07-16):** Section A's Cell 25 defines the
  two-site `ν1[λ1_,λ2_][u_] := (u-θ1-hλ1)(u-θ2-hλ2)`; Section E's Cell 58 independently defines a
  single-site `ν1[λ_][u_] := (u-θ1-hλ)`. These coexist as separate DownValues on the same head
  (Wolfram dispatches by argument-count pattern), which is not a conflict, but a reader searching
  the notebook for `ν1` will find two differently-shaped formulas under the same name and should
  check which arity is meant.
- **`Qθ` defined twice with the same formula:** Section A (Cell 21) and Section B (Cell 30, after
  its own `ClearAll[Qθ]`) both define `Qθ[u_] := (u-θ1)(u-θ2)`. Currently harmless since they agree,
  but a future edit to one without the other will silently diverge. `Qθ1[u]=(u-θ1)` (Cell 21) is
  **no longer referenced anywhere** in the live notebook as of the 2026-07-16 fix — Section E's
  `xL1` used to call it directly but now uses its own single-site `ν1[λ][u]` instead (numerically
  identical, a notational-parity rename, not a behavior change — see Section E).
- **`x2`'s normalization was fixed 2026-07-16 (do not revert):** the earlier denominator divided by
  the symmetric `Qθ(u)=(u-θ1)(u-θ2)` evaluated at a single argument shifted by $-\lambda_1h$ for
  *both* the `{s1,·}` and `{·,s2}` factors — silently applying site 1's shift to the $\theta_2$
  factor that should have carried site 2's own $\lambda_2$. This produced an apparently nontrivial
  "site-2 passthrough eigenvalue" in the old Section E, which was actually the bug's signature, not
  a genuine finding (see Section E). The fix (Section A's `ν1[λ1,λ2][u]`, with independently
  shifted per-site factors) makes the Section E factorization check come out exactly $1$ — if this
  denominator is ever "simplified" back to a `Qθ`-based form, re-run Section E's factorization check
  (Cell 60) to confirm it doesn't silently regress.
- **Memoization caching stale results:** several routines (`ps`, `dim`, `J0`/`Jp`/`Jm`/`JJ`, `T`,
  `x2`, `t2Coeffs`, `TauEigensystem`, `TauEigensystemLeft`, `QSolve`, `xL1`, `Nn1`) are
  self-memoizing (`f[x_]:=f[x]=...`). Per the project-wide convention (`CLAUDE.md`), after fixing
  any upstream definition, re-run the defining cell and everything downstream that used the stale
  cache — a partial re-run will not clear old memoized values. In particular, `x2` is memoized, so
  the 2026-07-16 fix required re-evaluating Cell 26 and everything downstream of it (Section E's
  factorization checks, Section F's `FLdet`/dictionary) rather than just editing the definition.
- **Precision-tracked exact zeros before `NullSpace`:** see the `QSolve` gotcha above — always
  `Chop` a numeric matrix before `NullSpace` when its singularity is expected by construction rather
  than relying on `Tolerance` alone.
- **`h=1`, high-precision twist:** Cell 1 sets `κ1 = N[Exp[I Zeta[3]], 30]` (30-digit precision,
  complex, on the unit circle) with `κ2=1/κ1=Conjugate[κ1]`. Numeric checks throughout rely on this
  tracked precision (residuals reported at $10^{-20}$ to $10^{-34}$ are relative to this ~30-digit
  budget, not to machine precision) — don't casually substitute lower-precision numeric twist values
  without re-checking tolerance thresholds (e.g. the hard `10^{-15}`/`10^{-10}`/`10^{-12}` asserts
  scattered through Sections B–F.
- **`normRatio` (Section D) is scoped to a single fixed representation:** it divides by the global
  `G21[[i,i]]` and looks up its index via the global `states21`, so it is only valid for
  $(\lambda_1,\lambda_2)=(2,1)$ — not a general $(\lambda_1,\lambda_2)$-parametrized routine, despite
  taking `[λ1,λ2]` as arguments.
- **`braket`/biorthogonality use a non-Hermitian bilinear pairing:** `PsiLeft[M,n].Psi[M',n']` with
  no complex conjugation anywhere, consistent with every other pairing in this notebook (`det`,
  `br`, `GramMatrix`) — this is a deliberate convention (the companion twist $G$ and Q-functions are
  genuinely non-Hermitian objects), not an oversight.
- **`gRot`'s `MatrixPower[·,0]` quirk (Section F):** in this kernel, `MatrixPower[singularMatrix,0]`
  does not auto-reduce to the identity matrix even for the literal integer `0` — it fires
  `MatrixPower::sing` and stays symbolically unevaluated, because `Ee[λ][a,b]` is singular for the
  raising/lowering generators used here. `gRot` works around this by splitting off the $m=0$
  identity term explicitly rather than summing `MatrixPower[·,m]` from `m=0`.
- **`FLdet`'s clean-polynomial property relies on prefactor cancellation:** `Q1` already carries
  `κ1^(u/h)`, so `k^(u/h) Q1[...]` is really `(k κ1)^(u/h) q_n(u)`, whose value at $u=\theta_1$ is a
  fractional power of $k\kappa_1$; `det1`'s division by `f/.u->θ1` is exactly what cancels this
  fractional prefactor, leaving `FLdet` as a clean polynomial in $k$ (not merely "the bracket applied
  to a `κ1`-twisted power") — this is why Section F uses `det1`, not the bare `br1`, as the
  functional object.
