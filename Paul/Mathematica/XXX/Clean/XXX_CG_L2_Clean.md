The routines below jointly solve $t_{1,1}(u)\Psi = \tau(u)\Psi$ and the Baxter TQ equation
$$\kappa_1\,a(u)\,q(u+h) + \kappa_2\,Q_\theta(u)\,q(u-h) = \tau(u)\,q(u)$$
for the two-site chain, in two stages: (1) `TauEigensystem` diagonalizes the *genuine* transfer
matrix directly (no oversized Baxter-matrix proxy, no post-hoc spurious-root filtering) to get
every $(M,n)$-indexed eigenvalue $\tau=\tau_0+J_n$; (2) `QSolve` then solves for the monic
degree-$M$ Q-function $q_n(u)$ as a linear `NullSpace` problem at the now-known $J_n$. This
supersedes an earlier (superseded, still in `Experiments/`) approach that diagonalized an
oversized $(M{+}1)$-dimensional Baxter matrix directly and could manufacture spurious extra roots.
See `Experiments/Baxter_Solver_L2.wb` for the full derivation writeup and validation/regression
suite (not duplicated into this Clean notebook). Section C below builds a full numeric verification
of orthogonality of the resulting `Q1` Q-functions; Section F builds a closed-form dictionary
relating them, functionally, to an operatorial overlap.

- **Purpose:** the Baxter-equation ingredients, defined locally so Section B is self-contained: the
  Bethe/Baxter "$a(u)$" coefficient, the $J$-independent part of the transfer-matrix eigenvalue
  $\tau_0$, the two-site $Q_\theta$ factor (redefinition of Section A's, see gotcha above), and a
  helper that evaluates an ascending-coefficient-list polynomial without an indeterminate $0^0$.
- **Signature:**
  - `aFun[őĽ1_,őĽ2_][u_] := (u-őł1-h őĽ1)(u-őł2-h őĽ2)`.
  - `Qőł[u_] := (u-őł1)(u-őł2)` ‚ÄĒ identical to Section A's, re-`ClearAll`'d and redefined.
  - `ŌĄ0[őĽ1_,őĽ2_,M_][u_] := (őļ1+őļ2)u^2 - (őł1+őł2)(őļ1+őļ2)u - u h(őļ1(őĽ1+őĽ2-M)+őļ2 M)` ‚ÄĒ note zero

  section stops at the pseudovacuum/covector construction). `a(u)=\prod_i(u-\theta_i-h\lambda_i)$,
  $Q_\theta(u)=\prod_i(u-\theta_i)$ is the standard XXX Baxter-equation form generalized to two
  inhomogeneous sites with a companion twist of eigenvalues $\kappa_1,\kappa_2$.

### `BaxterMatrix[őĽ1,őĽ2,M]`

- **Purpose:** the fixed linear operator, in the monic-degree-$M$-polynomial coefficient basis,
  whose nullspace at a known $J$ gives the Q-function's coefficients. Historically also used
  (superseded approach, in `Experiments/`) as an oversized eigenvalue problem; here it is used only
  as a linear operator via `QSolve`.
- **Signature:** `BaxterMatrix[őĽ1_,őĽ2_,M_]` builds, for $k=0,\dots,M$, the column

  $(M{+}2)$ coefficients vanish for every column ‚ÄĒ this is where a wrong `aFun`/`Qőł`/`ŌĄ0` or a
  wrong $ee_1=\lambda_1+\lambda_2-M$ convention would show up. The surviving degree-$0..M$
  coefficients of each column become the matrix's columns.

- **Depends on:** [`aFun`, `ŌĄ0`, `Qőł`](#afunőĽ1őĽ2u-ŌĄ0őĽ1őĽ2mu-qőłu-section-b-qpolyclistuu).

- **Verified by:** the hard in-function assert (no separate check cell in this notebook); Cell 29's

### `TauEigensystem[őĽ1,őĽ2]`

- **Purpose:** diagonalizes the *genuine* transfer matrix $t_{1,1}[\lambda_1,\lambda_2](u)$
  directly (not the oversized Baxter-matrix proxy), returning every $(M,n)$-indexed eigenstate with
  a hard genuineness residual check.
- **Signature:** `TauEigensystem[őĽ1_,őĽ2_] := TauEigensystem[őĽ1,őĽ2] = Module[...]` (memoized).
  Algorithm: since $C_0,C_1,C_2$ pairwise commute, diagonalizes a generic combination
  $C_0+rC_1$ (trying $r\in\{7/3,\pi,e,11/5,13/4,\sqrt2\}$ until the combination's eigenvectors have
  full rank $d=(\lambda_1{+}1)(\lambda_2{+}1)$ and $d$ distinct eigenvalues ‚ÄĒ guards against an
  accidental degeneracy at a specific $r$). For each eigenvector $v$: extracts
  $t_0=\frac{v\cdot C_0 v}{v\cdot v}$, $t_1=\frac{v\cdot C_1 v}{v\cdot v}$, and hard-`Abort`s if the
  residuals $\|C_0v-t_0v\|,\|C_1v-t_1v\|$ exceed $10^{-15}$ (this residual check *is* the
  genuineness test, replacing the old post-hoc spurious-state filter). $M$ is read off
  algebraically from $t_1$ via
  $$M=\frac{-(\theta_1+\theta_2)(\kappa_1+\kappa_2)-h\kappa_1(\lambda_1+\lambda_2)-t_1}{h(\kappa_2-\kappa_1)}$$
  (inverting $\tau_0$'s $u^1$ coefficient), hard-asserted to land within $10^{-10}$ of an integer;
  $J=t_0$ directly (since $\tau_0$'s $u^0$ coefficient is zero by construction). States within each
  $M$-sector are sorted ascending by $(\mathrm{Re}[J],\mathrm{Im}[J])$ to assign $n$, and each
  sector's count is hard-asserted to equal `dPred[M,őĽ1,őĽ2]` exactly.
- **Returns:** an `Association` keyed by `{M,n}` $\to$ `<|"J"->J_n, "Psi"->eigenvector normalized to
  last component 1|>`.
- **Depends on:** [`t2Coeffs`](#t2coeffsőĽ1őĽ2), [`dPred`](#dpredmőĽ1őĽ2).
- **Defined in:** Cell 31.
- **Paper reference:** none (Baxter/TQ machinery not yet in `paper/xxxCG.tex`).
- **Verified by:** the in-function hard asserts (rank/degeneracy resolution, residual $<10^{-15}$,
  integer $M$ within $10^{-10}$, per-sector count exactly matching `dPred`) constitute the check;
  a real failure aborts rather than silently returning wrong data. Independently,
  `Experiments/Baxter_Solver_L2.wb` Cells 10‚Äď11 run an *external* re-check for

### `QSolve[őĽ1,őĽ2][M,n]`

- **Purpose:** solves the Baxter TQ equation as a *linear* problem (not an eigenvalue problem, since
  $J_n$ ‚ÄĒ hence $\tau$ ‚ÄĒ is already known from `TauEigensystem`): finds the monic degree-$M$
  polynomial $q_n(u)$ with `(BaxterMatrix[őĽ1,őĽ2,M] - Jn¬∑Id).q_n = 0`.

  (singular by construction) and calls
  `NullSpace[Chop[mat, 10^-12], Tolerance -> 10^-10]`.
- **Returns:** the length-$(M{+}1)$ nullspace vector normalized so its last (top, $u^M$) coefficient
  is $1$ (monic), or `$Failed`/message on `degnull`/`lastzero`/`bnds`/`spurious` failure.
- **Depends on:** [`BaxterMatrix`](#baxtermatrixőĽ1őĽ2m), [`TauEigensystem`](#taueigensystemőĽ1őĽ2),

- **Defined in:** Cell 33.

  genuinely singular, but its would-be-zero entries come out as **precision-tracked** zeros. `Chop`

## Symbol reference ‚ÄĒ Section C (FSoV scalar product / orthogonality)

Section C ports the Functional Separation of Variables (FSoV) machinery from
`Experiments/FSoV_L2.wb` and uses it to numerically verify orthogonality/non-degeneracy of the
`Q1` Q-functions computed in Section B, over a full representation at once (all $M$-sectors,
including cross-$M$ pairs). See
`docs/superpowers/specs/2026-07-16-fsov-scalar-product-design.md` for the design writeup.

### `mu[őĽ1,őĽ2][i][u]`

- **Purpose:** the FSoV measure for site $i\in\{1,2\}$: a rational function with simple poles on
  that site's inhomogeneity tower $\theta_i+hk$, $k=0,\dots,\lambda_i$, carrying an extra
  cross-site oscillatory factor that vanishes the residue sum of `1` (needed for `br` to correctly
  isolate the $\theta_i$-tower contribution without a spurious constant term).
- **Signature:**
  `mu[őĽ1_,őĽ2_][1][u_] := Nn[őĽ1,őĽ2][1]/(Product[(u-őł1-h k),{k,0,őĽ1}]Product[(u-őł2-h k),{k,0,őĽ2}])(1-Exp[2 ŌÄ I/h (u-őł2)])`,
  symmetric for site 2 with `őł1`/`őł2` swapped in the oscillatory factor.
- **Returns:** a rational (times exponential) function of $u$.
- **Depends on:** [`Nn`](#nnőĽ1őĽ2i-nnőĽ1őĽ2i).
- **Defined in:** Cell 36.
- **Paper reference:** none yet ‚ÄĒ `paper/xxxCG.tex` does not state an FSoV scalar product; ported
  from the exploratory `Experiments/FSoV_L2.wb`.

### `Nn[őĽ1,őĽ2][i]`, `nn[őĽ1,őĽ2][i]`

- **Purpose:** normalization factors in the site-$i$ measure, fixing its overall residue sum so that
  `det` below has the right normalization to be a genuine scalar product.
- **Signature:**
  `Nn[őĽ1_,őĽ2_][1] := 1/(2 ŌÄ I)/(1-Exp[2 ŌÄ I/h(őł1-őł2)]) nn[őĽ1,őĽ2][1]`;
  `nn[őĽ1_,őĽ2_][1] := (-1)^(őĽ1+őĽ2-1)h^(őĽ1)Gamma[őĽ1+1] Product[őł2+k h-őł1,{k,0,őĽ2}]//Simplify`
  (symmetric for site 2).
- **Depends on:** nothing else project-defined.
- **Defined in:** Cell 37.

### `br[őĽ1,őĽ2][i][f]`

- **Purpose:** the bracket pairing for site $i$: the residue sum of $f\cdot\mu_i$ over all of site
  $i$'s poles ‚ÄĒ a contour integral around the whole $\theta_i$-tower.
- **Signature:**
  `br[őĽ1_,őĽ2_][1][f_] := 2 ŌÄ I Plus@@(Residue[f mu[őĽ1,őĽ2][1][u],{u,#}]&/@Table[őł1+h k,{k,0,őĽ1}])//Expand//Simplify`
  (symmetric for site 2). `f` is any function of `u` (typically a product of Q-functions).
- **Returns:** a symbolic/numeric value (whatever `f` evaluates to under the residue sum).
- **Depends on:** [`mu`](#muőĽ1őĽ2iu).
- **Defined in:** Cell 38.

### `det[őĽ1,őĽ2][f]`

- **Purpose:** the $2\times2$ Wronskian-type determinant built from `br[¬∑][f]` and `br[¬∑][u f]` at
  both sites ‚ÄĒ this **is** the FSoV/SoV scalar product. For $f=Q_A(u)Q_B(u)$ (a product of two
  Baxter Q-functions), `det[őĽ1,őĽ2][f]` realizes $\langle Q_A,Q_B\rangle$: it must vanish for $A\neq
  B$ and be nonzero for $A=B$.
- **Signature:**
  `det[őĽ1_,őĽ2_][f_] := 1/(f/.u:>őł1)/(f/.u:>őł2)/(őł2-őł1)({{br[őĽ1,őĽ2][1][f],br[őĽ1,őĽ2][1][u f]},{br[őĽ1,őĽ2][2][f],br[őĽ1,őĽ2][2][u f]}}//Det//Expand//Simplify)`.
- **Returns:** a scalar (the normalized $2\times2$ determinant of bracket values).
- **Depends on:** [`br`](#brőĽ1őĽ2if).

## Symbol reference ‚ÄĒ Section D (left eigenvectors and biorthogonality)

Section D is an **independent** construction (it never edits `TauEigensystem`): it diagonalizes
`Transpose[C0 + r C1]` for its own generic $r$, then matches each resulting left eigenvector to an
existing $\{M,n\}$ state by comparing eigenvalues. It exists to build $\langle\Psi|\Psi\rangle$ as a
genuine bilinear (no conjugation) pairing and compare it against Section C's FSoV scalar product
`det[Q,Q]`. See `docs/superpowers/specs/2026-07-16-fsov-left-eigenvector-ratio-design.md`; this is
exploratory in the sense that no predicted formula for `normRatio` exists yet in
`paper/xxxCG.tex` ‚ÄĒ the cell reports the ratio, it doesn't assert a value for it (though the
observed value is consistently $1$).

### `TauEigensystemLeft[őĽ1,őĽ2]`

## Symbol reference ‚ÄĒ Section F (functional CG-overlap dictionary)

Section F is the notebook's headline result: a closed dictionary equating a purely *functional*
object built from the numeric Baxter Q-functions (`FLdet`, a 1-site FSoV determinant of

### `gRot[őĽ][a,b][ŌÜ]`, `OP[őĽ1,őĽ2][a,b][M,n][ŌÜ]`, `genNontrivial`, `OPg[őĽ1,őĽ2][M,n][ŌÜ]`

- **Signature:**
  `gRot[őĽ_][a_,b_][ŌÜ_] := IdentityMatrix[dim[{őĽ,0}]] + Sum[ŌÜ^m/m! MatrixPower[Ee[őĽ][a,b],m],{m,1,őĽ}]`;
